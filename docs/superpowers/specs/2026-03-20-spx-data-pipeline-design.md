# SPX Data Pipeline Design

## 1. Overview

**Project**: QF5214 Data Engineering Course Project
**Goal**: Build a production-like SPX 500 data pipeline that simulates real financial data API behavior
**Data**: 20 years of OHLCV, fundamentals, and earnings call transcripts for ~818 SPX tickers

## 2. Architecture

```
Existing Dataset (CSV/PDF)
    ↓
DataProvider API
(Encapsulates raw data, simulates Yahoo Finance API)
    ↓
Bronze Layer (OLTP)
watchdog monitors landing_zone, raw data ingested
    ↓
ELT Pipeline
Bronze → Silver transformation (DuckDB SQL + Python)
    ↓
Silver Layer
Clean data (Parquet, partitioned by date/ticker) + Sentiment Analysis
    ↓
Gold Layer (OLAP)
DuckDB analytical views + Streamlit Dashboard
```

## 3. Core Components

| Component | Responsibility |
|-----------|---------------|
| **DataProvider API** | Read from CSV/PDF, simulate financial API (return data if exists, empty if not) |
| **Simulator** | Virtual clock advances, call DataProvider by date, emit to landing_zone only if data exists |
| **Ingestion Engine** | watchdog monitors landing_zone, batch ingest to Bronze on file detection |
| **ELT Job** | Read from Bronze → Transform → Write to Silver (Unpivot, Parquet partitioning, sentiment analysis) |
| **OLAP Views** | Sentiment-price correlation analysis + Multi-dimensional statistics |

## 4. DataProvider API

Simulates Yahoo Finance API behavior:

```python
# Price: data exists → DataFrame, no data → empty DataFrame
df = get_price(ticker: str, date: str) -> pd.DataFrame

# Fundamentals: data exists → dict, no data → empty dict
data = get_fundamentals(ticker: str, freq: str) -> dict
# freq: 'annual' or 'quarterly'

# Transcript: data exists → bytes, no data → FileNotFoundError
pdf_bytes = get_transcript(ticker: str, date: str) -> bytes

# List available transcripts
transcripts = list_transcripts(ticker: str = None, year: int = None) -> list

# Get trading dates in range
dates = get_trading_dates(start_date: str, end_date: str) -> list

# Get all available tickers
tickers = get_ticker_list() -> list

### Error Handling

| Error | Behavior |
|-------|----------|
| Data exists but unreadable (corrupted file) | Raise `DataIntegrityError`, log to audit |
| Ticker not found in dataset | Raise `ValueError` |
| Date format invalid | Raise `ValueError` with descriptive message |
| PDF exists but corrupted | Return empty bytes, log warning to audit |

## 5. Simulator (Virtual Clock)

```
Virtual Clock: 2024-01-15
    │
    ├─→ get_price('AAPL', '2024-01-15') → DataFrame → emit to output/landing_zone/prices/price_2024-01-15.csv
    ├─→ get_transcript('AAPL', '2024-02-01') → no such date → skip
    └─→ get_fundamentals('AAPL', '2024-01-31') → no such date → skip

Virtual Clock: 2024-01-16
    └─→ get_price('AAPL', '2024-01-16') → empty (weekend) → skip (no file emitted)
```

Each data source operates independently based on its own date. Data is emitted only if it exists.

### Landing Zone Emit Conventions

| Data Type | File Path | Format |
|-----------|-----------|--------|
| Price | `landing_zone/prices/price_{YYYY-MM-DD}.csv` | CSV, one row per ticker |
| Fundamentals | `landing_zone/fundamentals/{YYYY-MM-DD}/TICKER_type_freq.csv` | CSV, original format |
| Transcript | `landing_zone/transcripts/TICKER_{YYYY-MM-DD}.pdf` | PDF binary |

### Backfill Mode

For initial historical data load, simulator supports batch backfill:

```bash
# Backfill all 20 years of data at once (no sleep between batches)
python simulator.py --mode backfill --start 2004-01-02 --end 2024-12-30

# Backfill with delay (for observing pipeline behavior)
python simulator.py --mode backfill --start 2004-01-02 --end 2024-12-30 --delay 0.1
```

Watermark tracking: Simulator records last processed date in `output/.watermark` to support resume after interruption.

## 6. Bronze Layer (OLTP)

### Landing Zone Structure

```
output/landing_zone/
├── prices/
│   └── price_YYYY-MM-DD.csv          # Daily price batch
├── fundamentals/
│   └── YYYY-MM-DD/
│       └── TICKER_type_freq.csv      # Financial report for that date
└── transcripts/
    └── TICKER_YYYY-MM-DD.pdf         # Earnings transcript PDF
```

### Bronze Tables

```sql
-- Ingestion Audit (also serves as error log for DataProvider errors)
-- status values: 'SUCCESS', 'FAILED', 'PARTIAL'
CREATE TABLE ingestion_audit (
    id            BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    source        VARCHAR(50),      -- 'price' / 'fundamental' / 'transcript'
    ticker        VARCHAR(20),
    market_date   DATE,             -- The data's market date (from content/filename)
    received_at   TIMESTAMP DEFAULT NOW(),
    file_hash     VARCHAR(64),
    status        VARCHAR(20) DEFAULT 'SUCCESS',
    error_message VARCHAR(500)       -- NULL if SUCCESS, error details if FAILED
);

-- Raw Price Stream
CREATE TABLE raw_price_stream (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticker      VARCHAR(20),
    date        DATE,               -- Market date
    open        DECIMAL(18, 6),
    high        DECIMAL(18, 6),
    low         DECIMAL(18, 6),
    close       DECIMAL(18, 6),
    adj_close   DECIMAL(18, 6),
    volume      BIGINT,
    received_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(ticker, date)
);

-- Raw Fundamental Index
CREATE TABLE raw_fundamental_index (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticker      VARCHAR(20),
    report_type VARCHAR(30),        -- 'income_annual', 'balance_quarterly', etc.
    fiscal_date DATE,               -- The fiscal period date
    file_path   VARCHAR(500),
    received_at TIMESTAMP DEFAULT NOW()
);

-- Raw Transcript Index
CREATE TABLE raw_transcript_index (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticker      VARCHAR(20),
    event_date  DATE,               -- Earnings announcement date
    pdf_path    VARCHAR(500),
    text_hash   VARCHAR(64),
    received_at TIMESTAMP DEFAULT NOW()
);
```

## 7. ELT Pipeline

Type: **ELT** (Load raw first, then Transform using DuckDB SQL)

```
Bronze (raw data) ──ELT──→ Silver (clean Parquet)
```

### Transform Logic

| Source | Transform | Output |
|--------|-----------|--------|
| raw_price_stream | Partition by date, validate types | silver_price (Parquet, date=YYYY-MM-DD/) |
| raw_fundamental CSV | Unpivot (dates as rows), normalize | silver_fundamentals (Parquet, ticker=XXX/) |
| raw_transcript_index | Extract text from PDF | silver_transcript_text |
| raw_transcript_index | Compute sentiment via TextBlob | silver_transcript_sentiment |

### Idempotency

- Same market_date data is not written twice
- Each batch insert is atomic (success/failure)
- Audit trail with received_at and market_date

## 8. Silver Layer

### Output Structure

```
silver/
├── price/
│   └── date=2004-01-02/data.parquet
│   └── date=2004-01-05/data.parquet
│   └── ...
├── fundamentals/
│   └── ticker=AAPL/data.parquet
│   └── ticker=MSFT/data.parquet
│   └── ...
├── transcript_text/
│   └── ticker=AAPL/date=2024-02-01/content.txt
│   └── ...
└── transcript_sentiment/
    └── ticker=AAPL/date=2024-02-01/sentiment.parquet
```

### Sentiment Analysis

- Use TextBlob to compute polarity score (-1 to +1)
- Store as column in silver_transcript_sentiment
- **Trigger**: Asynchronous — when ELT job processes a new transcript PDF, it extracts text and computes sentiment as part of the transform step (not blocking ingestion)
- **Error handling**: If TextBlob fails on a PDF, log warning and store NULL sentiment (don't fail the whole batch)

### Data Quality Validation

Basic validation rules applied during Bronze → Silver transform:
- `price > 0`
- `volume >= 0`
- `high >= low` (OHLC consistency)
- `high >= open, high >= close, low <= open, low <= close`
- Invalid rows logged to `silver_quality_issues` table, not dropped

## 9. Gold Layer (OLAP)

### Views

```sql
-- Sentiment-Price Correlation
CREATE VIEW v_sentiment_price_analysis AS
SELECT
    p.ticker,
    p.date AS price_date,
    p.close,
    t.sentiment_polarity,
    t.sentiment_subjectivity,
    LAG(p.close, 1) OVER (PARTITION BY p.ticker ORDER BY p.date) AS prev_close,
    LAG(p.close, 5) OVER (PARTITION BY p.ticker ORDER BY p.date) AS close_5d_ago,
    LAG(p.close, 20) OVER (PARTITION BY p.ticker ORDER BY p.date) AS close_20d_ago,
    f.gics_sector
FROM silver_price p
LEFT JOIN silver_transcript_sentiment t ON p.ticker = t.ticker AND p.date = t.event_date
LEFT JOIN silver_fundamentals f ON p.ticker = f.ticker;

-- Multi-Dimensional Statistics
CREATE VIEW v_sector_stats AS
SELECT
    f.gics_sector,
    EXTRACT(YEAR FROM p.date) AS year,
    EXTRACT(QUARTER FROM p.date) AS quarter,
    AVG(p.volume) AS avg_volume,
    STDDEV(p.volume) AS volume_std,
    AVG(f.pe_ratio) AS avg_pe,
    COUNT(DISTINCT p.ticker) AS ticker_count
FROM silver_price p
JOIN silver_fundamentals f ON p.ticker = f.ticker
GROUP BY f.gics_sector, EXTRACT(YEAR FROM p.date), EXTRACT(QUARTER FROM p.date);
```
